# Geometry OS -- WebAssembly Browser Port

Run the Geometry OS VM entirely in your browser. No server, no install.

## Quick Start

```bash
# Build the WASM package
wasm-pack build --target web

# Serve locally (any static file server)
cd wasm
python3 -m http.server 8080

# Open http://localhost:8080 in your browser
```

## Architecture

```
wasm/
├── Cargo.toml          # WASM crate config
├── src/lib.rs          # wasm-bindgen API (GeometryOS class)
├── index.html          # Demo page with built-in programs
├── .gitignore          # Ignore pkg/ and target/
└── pkg/                # Generated by wasm-pack (gitignored)
```

## JavaScript API

```js
import init, { GeometryOS } from './pkg/geometry_os_wasm.js';
await init();

const geo = new GeometryOS('canvas-id');
geo.load(source);        // Assemble and load a program
geo.tick();              // Run until FRAME or HALT
geo.step();              // Single step
geo.reset();             // Reset VM
geo.is_halted();         // Check if VM stopped
geo.get_reg(n);          // Read register (0-31)
geo.get_pc();            // Read program counter
geo.key_press(code);     // Set keyboard input
geo.key_clear();         // Clear keyboard input
geo.get_beep();          // Get pending beep [freq, dur] or null
geo.peek(addr);          // Read RAM
geo.poke(addr, value);   // Write RAM
```

## Key Features

- Full VM: all 41 opcodes work in the browser
- 256x256 pixel canvas with nearest-neighbor scaling
- Audio beep via Web Audio API
- Keyboard input for interactive programs
- 11 built-in demo programs
- Custom program editor
- 136K WASM binary (optimized)

## Limitations

- BEEP uses Web Audio API instead of aplay (different latency)
- Save/load (F7) not available in browser (no filesystem)
- Self-hosting ASM opcode works but limited by browser memory
- No clipboard paste (no native integration)
