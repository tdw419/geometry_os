  <name>Task 1: Add HTTP boot support to PixelRTSServer</name>
  <files>systems/pixel_compiler/serve/server.py</files>
  <action>
Update PixelRTSServer to support HTTP boot:

1. Add HTTP port constant:
   ```python
   HTTP_PORT = 8080
   ```

2. Update __init__ to accept HTTP options:
   ```python
   def __init__(
       self,
       rts_png_path: str,
       interface: Optional[str] = None,
       verbose: bool = False,
       enable_http: bool = False,
       http_port: int = HTTP_PORT
   ):
   ```

3. Add ServerStatus field:
   ```python
   http_running: bool = False
   ```

4. Add instance variable:
   ```python
   self._http_server = None
   self._enable_http = enable_http
   self._http_port = http_port
   ```

5. Update prepare_boot_files() to generate HTTP boot files when enable_http=True:
   - Call PXEConfig.ensure_ipxe_boot_files(tftp_root) to ensure undionly.kpxe exists
   - Generate boot.ipxe script using PXEConfig.generate_ipxe_script()
   - Generate pxelinux.cfg using generate_ipxe_chainload_config() instead of generate_default_config()
   - Write boot.ipxe to tftp_root (same directory as kernel/initrd from Phase 6)
   - Note: HTTP server serves from tftp_root, so all boot files are available via both TFTP and HTTP

6. Update start() to start HTTP server when enable_http=True:
   ```python
   if self._enable_http:
       from systems.pixel_compiler.serve.http_server import HTTPBootServer, HTTPConfig
       http_config = HTTPConfig(
           host=self.network_config.ip_address,
           port=self._http_port,
           root_dir=str(tftp_root)
       )
       self._http_server = HTTPBootServer(config=http_config)
       await self._http_server.start()
       self.status.http_running = True
   ```

7. Update stop() to stop HTTP server:
   ```python
   if self._http_server:
       await self._http_server.stop()
       self._http_server = None
       self.status.http_running = False
   ```

8. Update _print_startup_info() to show HTTP status:
   ```python
   print(f"  HTTP Server: port {self._http_port} ({'running' if self.status.http_running else 'not started'})")
   ```

9. Service startup order: TFTP -> DHCP proxy -> HTTP -> NBD
   (HTTP after DHCP so clients can start chainloading)

10. Error handling: If HTTP fails, continue with TFTP-only boot (graceful degradation)
