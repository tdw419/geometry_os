  <name>Task 1: Add iPXE chainload configuration to PXEConfig</name>
  <files>systems/pixel_compiler/serve/pxe_config.py</files>
  <action>
Add iPXE chainload support to PXEConfig class:

1. Add class constant for iPXE boot filename:
   - IPXE_BOOT_FILE = "undionly.kpxe" (standard iPXE UNDI driver)

2. Add method `generate_ipxe_chainload_config(server_ip: str, http_port: int = 8080, nbd_port: int = 10809) -> str`:
   - Generates pxelinux.cfg/default that chainloads iPXE
   - Uses KERNEL directive to load undionly.kpxe
   - Format:
     ```
     DEFAULT ipxe
     TIMEOUT 50
     PROMPT 0

     LABEL ipxe
         KERNEL undionly.kpxe
     ```
   - Note: iPXE will then fetch boot.ipxe from HTTP server (Plan 02)

3. Update `generate_ipxe_script()` method (already exists):
   - Ensure it generates proper iPXE script with HTTP URLs
   - Format:
     ```
     #!ipxe
     kernel http://${server_ip}:${http_port}/vmlinuz initrd=initrd.img nbdroot=${server_ip}:${nbd_port} root=/dev/nbd0
     initrd http://${server_ip}:${http_port}/initrd.img
     boot
     ```

4. Add method `get_ipxe_boot_files() -> List[str]`:
   - Returns list of files needed for iPXE boot: ["undionly.kpxe"]

5. Add docstrings explaining the chainload flow:
   - Legacy PXE loads undionly.kpxe via TFTP
   - iPXE executes and fetches boot.ipxe via HTTP
   - boot.ipxe contains kernel/initrd URLs via HTTP

Do NOT modify ProxyDHCP or TFTPServer - those remain unchanged.
</action>
