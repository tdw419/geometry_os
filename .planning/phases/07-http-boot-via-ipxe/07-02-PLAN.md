  <name>Task 1: Create HTTPBootServer class with byte-range support</name>
  <files>systems/pixel_compiler/serve/http_server.py</files>
  <action>
Create HTTPBootServer class following the asyncio pattern from TFTPServer:

1. Create HTTPConfig dataclass:
   ```python
   @dataclass
   class HTTPConfig:
       host: str = "0.0.0.0"
       port: int = 8080
       root_dir: str = ""  # Directory containing boot files
   ```

2. Create HTTPBootServer class:
   - Uses asyncio.start_server() for TCP connections
   - Handles HTTP/1.1 GET requests
   - Serves files from root_dir
   - Path traversal protection (reject ".." in paths)

3. Implement HTTP request parsing:
   - Parse GET method and path
   - Parse Range header for partial content (format: "bytes=start-end")
   - Parse Connection header for keep-alive

4. Implement HTTP response building:
   - 200 OK for full file
   - 206 Partial Content for range requests
   - 404 Not Found for missing files
   - 400 Bad Request for invalid paths
   - 416 Range Not Satisfiable for invalid ranges

5. Required headers:
   - Content-Type (application/octet-stream for binaries, text/plain for .ipxe)
   - Content-Length
   - Content-Range for partial content
   - Accept-Ranges: bytes (advertise range support)

6. Example response for range request:
   ```
   HTTP/1.1 206 Partial Content
   Content-Type: application/octet-stream
   Content-Length: 1024
   Content-Range: bytes 0-1023/5120
   Accept-Ranges: bytes

   [binary data]
   ```

7. Implement async methods:
   - `start()` - Start the HTTP server
   - `stop()` - Stop the HTTP server
   - `_handle_client(reader, writer)` - Handle individual connections
   - `_parse_range_header(range_header: str, file_size: int) -> Tuple[int, int]`

8. Add logging for requests (method, path, range, response code)

Do NOT use aiohttp or external HTTP frameworks - use raw asyncio for consistency with TFTPServer.
