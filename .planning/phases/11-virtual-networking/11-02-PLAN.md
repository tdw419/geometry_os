  <action>
Modify `_build_network_args()` method (around line 654) to handle socket modes:

Add new elif branches after the BRIDGE case:

```python
elif self.config.network_mode == NetworkMode.SOCKET_MCAST:
    # Multicast socket networking (no root required)
    socket_config = self.config.socket_config or VirtualNetworkConfig()
    vn = VirtualNetwork(config=socket_config)
    args.extend(vn.build_netdev_args(device_id="net0"))
elif self.config.network_mode == NetworkMode.SOCKET_STREAM:
    # Point-to-point socket networking (no root required)
    # For stream mode, we use a listen/connect pattern
    # By default, first VM listens, others connect
    socket_config = self.config.socket_config or VirtualNetworkConfig()
    vn = VirtualNetwork(config=socket_config)
    args.extend(vn.build_stream_args(device_id="net0"))
```

Note: SOCKET_STREAM needs a `build_stream_args()` method in VirtualNetwork that generates:
- For listener: `-netdev socket,id=net0,listen=:PORT`
- For connector: `-netdev socket,id=net0,connect=HOST:PORT`

For now, SOCKET_MCAST is the primary use case (mesh networking where all containers join same multicast group). SOCKET_STREAM can be a stub that raises NotImplementedError with a helpful message.

DO NOT break existing NONE, USER, TAP, BRIDGE cases.
