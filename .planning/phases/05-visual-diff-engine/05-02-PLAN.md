  <files>systems/pixel_compiler/pixelrts_diff.py</files>
  <action>
Add `_get_regions(self, diff_mask: np.ndarray, grid_size: int) -> list` method to PixelRTSDiffer class.

Implementation:
1. Import HilbertCurve from pixelrts_v2_core
2. Try/except import scipy.ndimage (graceful fallback if unavailable)
3. Calculate Hilbert order: `order = int(np.log2(grid_size))`
4. Create HilbertCurve instance and generate LUT
5. Create 2D boolean pixel_mask array (grid_size x grid_size)
6. For each True index in diff_mask:
   - Calculate pixel_idx = byte_idx // 4 (RGBA packing)
   - Look up (x, y) from Hilbert LUT
   - Set pixel_mask[y, x] = True (bounds check: x < grid_size and y < grid_size)
7. Use scipy.ndimage.label(pixel_mask) to find connected regions
8. For each labeled region (cap at 100):
   - Extract coordinates with np.where(labeled == i)
   - Calculate x_min, x_max, y_min, y_max
   - Calculate pixel_count
9. Return list of region dicts sorted by pixel_count descending

Each region dict contains:
- id: "R1", "R2", etc.
- x_min, x_max, y_min, y_max (int)
- pixel_count (int)
- byte_count (pixel_count * 4, approximate)

If scipy unavailable, return empty list with warning logged.
</action>
