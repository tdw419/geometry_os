Modify `SnapshotExporter` at `systems/pixel_compiler/boot/snapshot_exporter.py`:

1. **Add kernel/initrd extraction to _encode_to_rts method**:

   Before encoding the qcow2, extract the original kernel/initrd:
   ```python
   def _extract_original_boot_files(self) -> Tuple[Optional[bytes], Optional[bytes]]:
       """Extract original kernel/initrd from booted container via FUSE."""
       # Use boot_bridge._mount_helper to access mounted filesystem
       if not self.boot_bridge._mounted or not self.boot_bridge._mount_helper:
           return None, None

       mount_helper = self.boot_bridge._mount_helper
       kernel_path, initrd_path = mount_helper.discover_boot_files()

       kernel_data = None
       initrd_data = None

       if kernel_path:
           with open(kernel_path, 'rb') as f:
               kernel_data = f.read()

       if initrd_path:
           with open(initrd_path, 'rb') as f:
               initrd_data = f.read()

       return kernel_data, initrd_data
   ```

2. **Store kernel/initrd in committed file**:

   The challenge: qcow2 is already encoded as the "data" portion. We need to store kernel/initrd alongside it.

   **Solution**: Store kernel/initrd in metadata offsets (like original containers):
   ```python
   # In _encode_to_rts, after reading qcow2_data:
   kernel_data, initrd_data = self._extract_original_boot_files()

   # Build combined data: qcow2 + kernel + initrd
   # Store offsets in metadata so decoder knows where each piece is
   combined_data = bytearray(qcow2_data)

   offsets = {}
   if kernel_data:
       kernel_offset = len(combined_data)
       combined_data.extend(kernel_data)
       offsets["kernel"] = {
           "offset": kernel_offset,
           "size": len(kernel_data),
           "sha256": hashlib.sha256(kernel_data).hexdigest()
       }

   if initrd_data:
       initrd_offset = len(combined_data)
       combined_data.extend(initrd_data)
       offsets["initrd"] = {
           "offset": initrd_offset,
           "size": len(initrd_data),
           "sha256": hashlib.sha256(initrd_data).hexdigest()
       }

   # Encode combined data
   encoder = PixelRTSEncoder(mode="standard")
   png_data = encoder.encode(bytes(combined_data), metadata=metadata)

   # Add offsets to metadata
   metadata["offsets"] = offsets
   metadata["disk_size"] = len(qcow2_data)
   ```

3. **Update metadata structure**:
   ```python
   metadata = {
       "type": "vm-snapshot",
       "snapshot_tag": snapshot_tag,
       "source_format": "qcow2",
       "created_at": datetime.now().isoformat(),
       "disk_size": len(qcow2_data),
       "offsets": offsets,  # kernel and initrd offsets
       "has_kernel": kernel_data is not None,
       "has_initrd": initrd_data is not None,
   }
   ```

4. **Add progress stage for boot file extraction**:
   - Add `EXTRACTING_BOOT_FILES = "extracting_boot_files"` to ExportStage enum
   - Report progress before/after extraction

5. **Handle edge cases**:
   - If VM not booted via BootBridge (e.g., external QEMU), log warning and skip kernel/initrd
   - If kernel/initrd discovery fails, log warning but continue (disk-only commit)
   - If kernel/initrd too large (>100MB combined), log warning and skip to avoid huge files

Do NOT break existing export functionality. The kernel/initrd capture is additive.
