  <name>Task 1: Implement SignatureVerifier class</name>
  <files>systems/pixel_compiler/verification/signature_verifier.py</files>
  <action>
Create SignatureVerifier extending VerificationStep that:

1. Gets public key (precedence: env var PIXELRTS_PUBLIC_KEY > metadata["signature"]["public_key"])
   - Env var contains hex string (64 chars = 32 bytes)
   - If neither available, return SKIP with "No public key available"

2. Gets signature from metadata["signature"]["value"] (hex string, 128 chars = 64 bytes)
   - If missing, return SKIP with "File is not signed"

3. Gets data hash from metadata["hashes"]["data"] (hex string, 64 chars = 32 bytes)
   - This is the SHA256 hash already computed by PixelRTSMetadata
   - If missing, return FAIL with "Cannot compute data hash for verification"

4. Performs Ed25519 verification:
   ```python
   from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey
   from cryptography.exceptions import InvalidSignature

   public_key = Ed25519PublicKey.from_public_bytes(public_key_bytes)
   public_key.verify(signature_bytes, data_hash_bytes)  # data_hash is what was signed
   ```

5. Returns:
   - PASS with truncated public key in details (16 chars + "...")
   - FAIL with "Signature verification failed - file may be tampered"
   - SKIP for unsigned files or missing public key

Key implementation details:
- Sign the HASH (32 bytes), not the full data - research confirms this is correct
- Use bytes.fromhex() to convert hex strings to bytes
- Wrap verify() call in try/except InvalidSignature
- Follow ConsistencyVerifier pattern for metadata access
- Name property: "signature"
- Description: "Verifies Ed25519 cryptographic signature"
