  <action>
    Create ConsistencyVerifier that validates hash consistency:

    **systems/pixel_compiler/verification/consistency_verifier.py**

    ConsistencyVerifier class (extends VerificationStep):
    - name = "consistency"
    - description = "Verifies decoded data hash matches metadata hash"

    verify() method performs these checks:

    1. **Metadata Hash Presence Check**
       - Get metadata from context.get_metadata()
       - Check if metadata["hashes"] exists and has entries
       - WARNING if no hashes present (can't verify, but not a failure)
       - Return early with SKIP status if no hashes

    2. **Data Hash Computation**
       - Get png_data from context.png_data
       - Use PixelRTSDecoder to decode the data
       - Compute SHA256 using PixelRTSMetadata.hash_data(decoded_data)
       - Catch decode errors and return FAIL with specific message

    3. **Hash Comparison**
       - For each hash in metadata["hashes"]:
         - Compare computed hash against stored hash
         - Track which hashes match/mismatch
       - Primary check: metadata["hashes"]["data"] or metadata["hashes"]["combined"]
       - FAIL if hash mismatch with both computed and expected in message
       - PASS if all hashes match

    4. **Result Details**
       - Include in details dict:
         - computed_hash: str
         - expected_hash: str (from metadata)
         - data_size: int (bytes decoded)
         - decode_time_ms: float

    Handle these edge cases:
    - Missing hashes in metadata (SKIP with message)
    - Decode failure (FAIL with decode error)
    - Multiple segment hashes (check each, report all)

    Use existing PixelRTSDecoder.decode() and PixelRTSMetadata.hash_data().
  </action>
