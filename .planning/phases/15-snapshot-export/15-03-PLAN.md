Create new file systems/pixel_compiler/boot/snapshot_exporter.py with:
- ExportStage enum (IDLE, COMMITTING, ENCODING, COMPLETE, FAILED)
- ExportProgress dataclass (stage, started_at, completed_at, message, bytes_processed, total_bytes)
- ExportResult dataclass (success, output_path, size_bytes, error_message, progress)
- SnapshotExporter class with:
  - __init__(boot_bridge, progress_callback=None)
  - _update_progress(stage, message, bytes_processed, total_bytes)
  - _encode_to_rts(qcow2_path, output_path) - read qcow2, encode via PixelRTSEncoder, write .rts.png
  - export(output_path, tag=None, timeout=600) - main pipeline

Pipeline in export():
1. Create temp directory for intermediate files
2. Call SnapshotCommitter.commit() to extract qcow2 -> update progress to COMMITTING
3. Read extracted qcow2 file -> update progress to ENCODING
4. Encode via PixelRTSEncoder.encode() with vm-snapshot metadata type
5. Write output .rts.png file
6. Cleanup temp files
7. Return ExportResult

For large files (>100MB), read and encode in chunks to avoid memory exhaustion.
Use PixelRTSEncoder in standard mode for qcow2 binary data.
