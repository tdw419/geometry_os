Create SegmentIntegrityChecker that verifies individual segments (kernel, initrd, disk) using range-based decode for efficiency.

Purpose: Enable fast verification of individual segments without requiring full container decode. This is critical for large files where full decode takes ~5 seconds but range decode is ~1ms.

Output: SegmentIntegrityChecker extending VerificationStep with per-segment verification capability.
</objective>

  <name>Task 1: Implement SegmentIntegrityChecker</name>
  <files>systems/pixel_compiler/verification/segment_integrity_checker.py</files>
  <action>
Create SegmentIntegrityChecker class extending VerificationStep.

Key implementation details:
1. **Class structure:**
   - name property: "segment_integrity"
   - description property: "Verifies individual segment integrity using range-based decode"
   - Optional `segment_name` parameter in constructor (None = verify all segments)

2. **verify() method logic:**
   - Get metadata from context
   - Check for `metadata["offsets"]` - if missing, return SKIP
   - For each segment in offsets (or just the specified segment if segment_name given):
     - Get offset and size from `metadata["offsets"][segment_name]`
     - Get expected hash from `metadata["hashes"][segment_name]` (if missing, SKIP that segment)
     - Call `decoder.decode_range(png_data, offset, size)` to get segment data
     - Compute SHA256 using `PixelRTSMetadata.hash_data(segment_data)`
     - Compare with expected hash
   - Return StepResult with PASS/FAIL/SKIP per segment

3. **Return format:**
   - Single segment: Return StepResult with that segment's status
   - All segments: Return StepResult with combined status (FAIL if any fail, else PASS)
   - Include details dict with per-segment results

4. **Error handling:**
   - If metadata missing, return SKIP with message
   - If segment not in offsets, return SKIP for that segment
   - If hash not in hashes, return SKIP for that segment
   - If decode_range raises exception, return FAIL with error details

5. **Import pattern (follow consistency_verifier.py):**
   ```python
   from .base import VerificationStep
   from .result import StepResult, VerificationStatus
   from .context import VerificationContext
   from systems.pixel_compiler.pixelrts_v2_core import PixelRTSDecoder, PixelRTSMetadata
   ```

DO NOT implement full decode - only use decode_range() for efficiency.
DO NOT create new result types - reuse VerificationStatus and StepResult.
