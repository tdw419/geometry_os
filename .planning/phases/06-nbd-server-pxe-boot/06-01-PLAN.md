<task type="auto">
  <name>Task 2: Implement range decoding for pread()</name>
  <files>systems/pixel_compiler/serve/nbd_plugin.py</files>
  <action>
Implement efficient range decoding in the `pread()` method.

The key challenge: PixelRTS stores bytes in Hilbert curve order across RGBA channels. To read a byte range [offset, offset+count), we need to:

1. Calculate which pixels contain the requested bytes
2. Decode only those pixels (not the entire image)
3. Extract the correct byte range from decoded data

Implementation approach:
- Use PixelRTSDecoder's internal methods for Hilbert mapping
- For a read at `offset` with `count` bytes:
  - First pixel: pixel_idx = offset // 4
  - Last pixel: pixel_idx = (offset + count - 1) // 4
  - Decode pixels from first to last (inclusive)
  - Extract bytes [offset % 4 : (offset % 4) + count]

If PixelRTSDecoder doesn't have a `decode_range()` method, add one to pixelrts_v2_core.py:
```python
def decode_range(self, png_bytes: bytes, start_byte: int, count: int) -> bytes:
    """Decode only a byte range from the PNG data."""
```

This is NBD-03 requirement: range decoding for memory-efficient serving of large containers.
