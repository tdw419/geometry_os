Create a TFTP server for serving boot files (pxelinux.0, kernels, initrds).

Implementation requirements:
1. Create `systems/pixel_compiler/serve/tftp_server.py`:
   - `TFTPServer` class with async design
   - Listen on UDP port 69 (standard TFTP port)
   - Support RRQ (Read Request) opcode
   - Implement TFTP protocol:
     - Block-based transfer (512 bytes per block default)
     - ACK handling for each block
     - Error handling for missing files
     - Timeout and retransmission support

TFTP protocol basics:
- Opcode 1 = RRQ (filename, mode)
- Opcode 2 = WRQ (not needed, read-only)
- Opcode 3 = DATA (block#, data)
- Opcode 4 = ACK (block#)
- Opcode 5 = ERROR (code, message)

Key methods:
- `start()` - Start listening on port 69
- `stop()` - Stop the server
- `set_root_dir(path)` - Set directory to serve files from
- `handle_rrq(filename, client_addr)` - Handle read request

Use asyncio for async UDP handling. Support concurrent transfers to multiple clients.
