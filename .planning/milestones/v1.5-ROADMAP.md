# Milestone v1.5: Commit to File

**Status:** ✅ SHIPPED 2026-03-09
**Phases:** 15-17
**Total Plans:** 9

## Overview

Users can commit running containers to new portable .rts.png files that preserve all disk and memory state. This milestone enables the complete container lifecycle: boot → modify → commit → boot again.

## Phases

### Phase 15: Snapshot Export

**Goal**: Users can export a VM snapshot to a standalone qcow2 file with streaming encoding.
**Depends on**: Phase 14 (VMSnapshotManager exists)
**Requirements**: COMMIT-01, COMMIT-02, COMMIT-03
**Plans**: 3 plans

Plans:
- [x] 15-01: QemuImgWrapper for qemu-img extract operations
- [x] 15-02: SnapshotCommitter with VM pause/resume orchestration
- [x] 15-03: SnapshotExporter with PixelRTS encoding and progress

**Key Components:**
- `QemuImgWrapper` - Extracts QEMU internal snapshots via `qemu-img convert -l`
- `SnapshotCommitter` - Orchestrates VM pause → snapshot → extract → resume
- `SnapshotExporter` - Full pipeline with PixelRTS encoding

### Phase 16: Boot from Committed

**Goal**: Committed .rts.png files boot correctly with preserved state.
**Depends on**: Phase 15
**Requirements**: BOOT-COMMIT-01, BOOT-COMMIT-02, BOOT-COMMIT-03, COMMIT-04
**Plans**: 4 plans (3 complete + 1 gap closure)

Plans:
- [x] 16-01: CommittedFileBooter class for vm-snapshot boot
- [x] 16-02: Kernel/initrd preservation in committed files
- [x] 16-03: CLI integration and verification
- [x] 16-04: Gap closure: Fix kernel/initrd extraction from committed files

**Key Components:**
- `CommittedFileBooter` - Detects vm-snapshot type, extracts qcow2/kernel/initrd
- Combined data encoding with offset metadata
- CLI boot command auto-detection routing

### Phase 17: CLI Integration

**Goal**: Users can commit containers with a single CLI command.
**Depends on**: Phase 16
**Requirements**: CLI-COMMIT-01, CLI-COMMIT-02, CLI-COMMIT-03
**Plans**: 2 plans

Plans:
- [x] 17-01: CLI commit command with --snapshot and --no-verify flags
- [x] 17-02: End-to-end verification checkpoint

**Key Components:**
- `cmd_commit()` - CLI command with container validation via MultiBootManager
- Progress output with stage tracking (COMMITTING, ENCODING, VERIFYING)
- Flags: --snapshot, --no-verify, --timeout, --quiet, --verbose

---

## Milestone Summary

**Key Decisions:**
- `qemu-img convert -l <snapshot_tag>` for snapshot extraction (preserves disk state)
- VM pause/resume during commit for state consistency
- Combined data encoding: qcow2 + kernel + initrd with offset metadata
- `ContainerType` enum for vm-snapshot detection
- CLI boot command auto-routing based on container type

**Issues Resolved:**
- Fixed kernel/initrd extraction from combined binary data (gap closure 16-04)
- Added disk_size trimming for qcow2 extraction
- Binary extraction with hash verification

**Technical Debt:**
- Large file chunked encoding not implemented (optimization opportunity)
- Legacy helper methods have minor field name issues (non-blocking)
