# Milestone v1.0: PixelRTS Boot Improvement

**Status:** ✅ SHIPPED 2026-02-27
**Phases:** 1-4
**Total Plans:** 23

## Overview

PixelRTS Boot Improvement transforms PixelRTS from a novel encoding format into a practical, production-ready OS boot and distribution system. The roadmap progresses from demonstrating PixelRTS's unique visual analysis advantage (Phase 1), to enabling direct PNG booting without extraction (Phase 2), to practical applications with visual installer (Phase 3), and finally to a polished visual catalog experience (Phase 4).

## Milestone Summary

**Key Decisions:**
- FUSE filesystem for direct boot (clean integration with existing tools)
- Vision model for verification (unique advantage of visual format)
- Phase 1: Vision analysis first (demonstrates core differentiator)
- Rich library for terminal output (graceful fallback to plain text)
- Lazy unmount with fusermount -uz for stuck mount recovery
- KVM acceleration enabled by default for <10% overhead
- InstallEngine verifies hash BEFORE writing to disk
- Singleton pattern via get_catalog_server() for shared server instance

**Issues Resolved:**
- Architecture validation bug in catalog scanner (nested metadata)
- Context overflow in 100+ phases (addressed with phase numbering)

**Technical Debt:** None

**Files Created:**
- `systems/pixel_compiler/catalog/` - Catalog components
- `systems/pixel_compiler/install/` - Install components
- `systems/pixel_compiler/boot/` - Boot components
- `systems/pixel_compiler/pixelrts_cli.py` - CLI entry point
- `bin/pixelrts` - CLI binary

**CLI Commands Shipped:**
- `pixelrts analyze <file.png>` - Vision analysis and verification
- `pixelrts boot <file.png>` - Boot .rts.png files with QEMU
- `pixelrts install <file.png> <target>` - Install .rts.png to disk image
- `pixelrts catalog` - Launch visual catalog server
