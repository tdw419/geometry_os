# Milestone v1.2: Network Boot

**Status:** ✅ SHIPPED 2026-03-09
**Phases:** 6-8.1
**Total Plans:** 10

## Overview

Boot PixelRTS containers over network (PXE/NBD) with bandwidth-efficient delta updates.

This milestone enables network booting of .rts.png containers with three key capabilities:
1. PXE boot via DHCP proxy + TFTP + NBD
2. HTTP boot for faster transfers via iPXE
3. Delta updates to download only changed bytes

## Milestone Summary

**Key Decisions:**

- Proxy DHCP mode (port 4011) to avoid conflicting with existing DHCP servers
- Range decoding in decoder class for memory-efficient serving
- Async orchestrator pattern for service coordination
- Graceful degradation if HTTP fails (continue TFTP-only)
- Decode/encode cycle for .rts.png files (checksums match decoded data)
- ByteFetcher protocol for remote region fetching
- Handler interface uses `handle(path, headers, writer) -> bool` pattern

**Issues Resolved:**

- DeltaHTTPHandler not wired to HTTPBootServer (Phase 8.1 gap closure)

**Issues Deferred:**

- UEFI vs BIOS PXE differences - needs real hardware testing
- Memory-mapped Hilbert LUT may need performance profiling
- Port 69 (TFTP) requires root privileges or CAP_NET_BIND_SERVICE

**Technical Debt:**

- Human verification needed: Full PXE boot test with real hardware

## CLI Commands Added

```bash
# Network boot
pixelrts serve <file.png>                    # Start network boot services
pixelrts serve <file.png> --http             # Enable HTTP boot (faster)
pixelrts serve <file.png> --http --delta     # Enable delta endpoints
pixelrts serve <file.png> --http --delta --delta-from <old.png>  # Generate initial delta

# Delta workflow
pixelrts delta <old.png> <new.png>           # Generate delta manifest
pixelrts delta <old.png> <new.png> -o manifest.json
pixelrts patch <base.png> <manifest.json>    # Apply delta patch
pixelrts patch <base.png> <manifest.json> -o output.png
pixelrts update <file.png> --server <url>    # Update via delta from server
```
