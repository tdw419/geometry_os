## v1.3 Requirements

### Core Multi-Boot

- [x] **MULTI-01**: User can boot multiple containers with single CLI command (`pixelrts boot a.rts.png b.rts.png`) — Complete (Phase 9)
- [x] **MULTI-02**: System allocates unique VNC ports and socket paths per container — Complete (Phase 9)
- [x] **MULTI-03**: Compensating transaction cleanup on partial boot failure — Complete (Phase 9)

### Boot Ordering

- [x] **ORDER-01**: User can designate primary container with --primary flag — Complete (Phase 10)
- [x] **ORDER-02**: Helper containers wait for primary to reach RUNNING state — Complete (Phase 10)
- [x] **ORDER-03**: Shutdown reverses boot order (helpers first, primary last) — Complete (Phase 10)

### Networking

- [x] **NET-01**: Containers can communicate with each other via QEMU socket networking — Complete (Phase 11)
- [x] **NET-02**: No root privileges required for inter-container networking — Complete (Phase 11)

### Status & Management

- [x] **STATUS-01**: User can view running containers with `pixelrts ps` — Complete (Phase 9)
- [x] **STATUS-02**: Status shows container states (launching/running/stopped) — Complete (Phase 9)
