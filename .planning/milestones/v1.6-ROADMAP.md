## Overview

Users can boot containers with `--ephemeral` flag to discard changes on exit. Complete ephemeral container lifecycle with temp file management, crash cleanup, and commit support.

## Phase

### Phase 18: Ephemeral Boot

**Goal**: Users can boot containers with `--ephemeral` flag to discard changes on exit.
**Depends on**: Phase 17
**Plans**: 8 plans

Plans:
- [x] 18-01: EphemeralBooter class with temp file management
- [x] 18-02: ContainerInfo ephemeral tracking
- [x] 18-03: CLI --ephemeral flag for single boot
- [x] 18-04: Multi-boot ephemeral support
- [x] 18-05: ps command [E] indicator
- [x] 18-06: Crash cleanup via atexit/signal handlers
- [x] 18-07: Commit support for ephemeral containers
- [x] 18-08: End-to-end verification checkpoint

**Success criteria:**
1. `pixelrts boot --ephemeral file.rts.png` starts container
2. Changes made inside container are NOT written to original file
3. Container exits cleanly with no leftover temp files
4. `pixelrts ps` shows `[E]` indicator for ephemeral containers
5. Ephemeral works with multi-boot
6. Ephemeral works with vm-snapshot type
7. Crash cleanup removes temp files
8. `pixelrts commit` can save ephemeral changes before exit

## Milestone Summary

**Key Decisions:**
- Delegation pattern: EphemeralBooter wraps BootBridge/CommittedFileBooter
- Temp file management: Copy source to temp dir on init, cleanup on stop/exit
- Crash safety: atexit registration + signal handlers ensure cleanup
- Commit support: original_path property allows commit to reference original metadata

**Issues Resolved:**
- Fixed multi-container ephemeral boot (gap found during audit)
- CLI _boot_multiple() now passes ephemeral flag to boot_all()

**Technical Debt:**
- 6 multi-boot mock tests failing (test setup issue, not implementation)
