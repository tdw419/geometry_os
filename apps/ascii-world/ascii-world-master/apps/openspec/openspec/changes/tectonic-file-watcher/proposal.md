# Tectonic File Watcher Proposal

## Summary

Connects real-time file system events to the Tectonic Gravity Engine, making the infinite map "alive" - files drift toward you as you use them.

## Motivation

The Geometry OS v1.2 vision describes "Semantic Gravity" where information is treated as a dynamic system of interacting masses. Currently, the Gravity Engine uses static mass based on file size. This misses the crucial real-time dimension: **files that are actively being used should be "hot" and drift toward the center.**

From the vision document:

> "The 'heat' is generated by computational intensity and neural activity. High-activity regions of the codebase become 'hot' and more fluid, allowing them to be refactored and moved more easily."

## Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    TECTONIC REAL-TIME GRAVITY PIPELINE                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   File System ──▶ FileWatcher ──▶ MassIntegrator ──▶ GravityEngine          │
│   (inotify)       (watchdog)       (event→mass)        (mass updates)        │
│                        │                 │                   │               │
│                        ▼                 ▼                   ▼               │
│                    FileEvent        MassDelta           orb.mass += delta    │
│                    (path, type)     (path, delta)                           │
│                                                                             │
│   Decay Loop ──────────────────────────────────────────────────────────────▶ │
│   (periodic mass reduction for inactive files)                              │
│                                                                             │
│   Glass Box ◀────────────────────────────────────────────────────────────── │
│   (files drift in real-time as they are used)                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Components

| Component | File | Purpose |
|-----------|------|---------|
| FileWatcher | `file_watcher.py` | Watch directory for file events using watchdog |
| MassIntegrator | `mass_integrator.py` | Convert events to mass deltas with decay |
| GravityEngine | `gravity_engine.py` | Accept real-time mass updates |
| TectonicUpdater | `tectonic_updater.py` | Wire components together with decay loop |
| Daemon | `daemon.py` | Enable/disable tectonic mode |

- Existing GravityEngine
- Existing Sisyphus Daemon

## Risk Mitigation

1. **Performance** - Event coalescing and throttling for high-activity periods
2. **Platform compatibility** - watchdog abstracts Linux inotify, macOS FSEvents, Windows
3. **Thread safety** - All mass updates go through GravityEngine's thread-safe methods
