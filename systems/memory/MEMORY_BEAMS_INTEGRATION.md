# Geometry OS: Memory Beams Integration (Hippocampus Path 2)

## Overview
Memory Beams provide a visual interface for Geometry OS's long-term semantic memory. By using the **Semantic Drag** gesture (Shift + Middle Mouse), users can connect their current spatial context to historically related "Thought Pixels" stored in the PostgreSQL vector database.

## Components

### 1. MemoryBeamRenderer (Frontend)
- **Location**: `systems/visual_shell/web/MemoryBeamRenderer.js`
- **Purpose**: High-performance PixiJS renderer for memory connections.
- **Visuals**: 
  - **Cyan Beams**: Glow lines connecting cursor to memory locations.
  - **Ghost Pixels**: Pulsing indicators at historical Hilbert coordinates.
  - **Context Previews**: Text labels showing the content of the memory.

### 2. Memory Visual Bridge (Middleware)
- **Location**: `systems/memory/memory_visual_bridge.py`
- **Purpose**: Translates between WebSocket (Visual Shell) and Unix Socket (Memory Daemon).
- **Functions**:
  - Map world coordinates to normalized Hilbert space.
  - Dispatch semantic queries to pgvector.
  - Stream results back to the browser in real-time.

### 3. Vector Memory Daemon (Backend)
- **Location**: `systems/memory/vector_memory_daemon.py`
- **Purpose**: Production-grade PostgreSQL Hippocampus.
- **Features**:
  - HNSW indexing for <50ms semantic search.
  - Persistent storage of every thought token generated by the OS.

## Usage: Semantic Drag
1. Ensure the memory daemons are running:
   ```bash
   ./systems/memory/vector_memory_daemon.py
   ./systems/memory/memory_visual_bridge.py
   ```
2. In the Visual Shell, **Hold Shift + Middle Mouse Button** and drag.
3. Observe Cyan beams connecting to related thoughts from past sessions.
